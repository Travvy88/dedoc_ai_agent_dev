\par \b Module Contract\/ \:\par \b Purpose\/ \: To define the base DedocError exception class — the root of the entire dedoc error hierarchy.

All specific error types inherit from this class, ensuring consistent error serialization and API representation. \par \b Scope\/ \: Base exception definition, error serialization (from_dict), version tracking. \par \b Input\/ \: Error context data (msg, filename, version, metadata, code). \par \b Output\/ \: Unified DedocError exception instances with API-friendly payloads. \par \b Links\/ \: [READS_DATA_FROM(7): dedoc.version] \par \b Invariants\/ \:

  • All DedocError subclasses inherit init signature and str pattern.
  • Every DedocError carries msg, msg_api, filename, version, metadata, code.
  • from_dict ALWAYS returns a valid DedocError instance (never None). \par \b Rationale\/ \:

Q: Why a rich error base class with msg_api, version, metadata instead of simple Exception? A: The dedoc API serves external clients — errors must be serializable to JSON with machine-readable fields (code, msg_api) for proper HTTP error responses. \par \b Changes\/ \:
