\par \b Module Contract\/ \:\par \b Purpose\/ \: To implement the OCREngine contract using pytesseract (Tesseract OCR) — the default engine for dedoc.

Converts pytesseract.image_to_data() dict output into engine-agnostic OCRResult. \par \b Scope\/ \: Tesseract-specific OCR: page recognition (PSM 3/4), cell recognition (PSM 6), confidence filtering, word-to-line grouping. \par \b Input\/ \: Config dict (ocr_conf_threshold), image (np.ndarray), language (str), psm_mode (int). \par \b Output\/ \: OCRResult with OCRLines and OCRWords. \par \b Links\/ \: [IMPLEMENTS(10): OCREngine; USES_API(9): pytesseract; USES_TYPE(9): OCRResult, OCRLine, OCRWord] \par \b Invariants\/ \:

  • recognize_page() ALWAYS calls pytesseract.image_to_data() with –psm {psm_mode}.
  • recognize_cells() ALWAYS calls pytesseract.image_to_data() with –psm 6.
  • Words with confidence < ocr_conf_threshold are filtered out.
  • Words with empty/whitespace text are filtered out.
  • Words are grouped by line_num → one OCRLine per line. \par \b Rationale\/ \:

Q: Why encapsulate all pytesseract logic here instead of keeping it in ocr_utils.py? A: Encapsulation isolates Tesseract-specific code (–psm flags, Output.DICT, confidence filtering) from the orchestration layer. New engines (EasyOCR, PaddleOCR) won't need to know anything about pytesseract. \par \b Changes\/ \:
