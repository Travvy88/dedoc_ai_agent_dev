\par \b Module Contract\/ \:\par \b Purpose\/ \: Define the LineMetadata schema — a Pydantic model for paragraph/line-level metadata (paragraph type, page number, line number) with extra attributes support.

\par \b Scope\/ \: Document line/paragraph metadata model. \par \b Input\/ \: None (standalone model with Extra.allow). \par \b Output\/ \: Pydantic BaseModel LineMetadata with paragraph_type, page_id, line_id + extra fields. \par \b Links\/ \: [USES_API(7): pydantic.BaseModel, pydantic.Extra] \par \b Invariants\/ \:

  • page_id >= 0.
  • paragraph_type is always a non-empty string (e.g., "raw_text", "header", "list_item").
  • Extra fields preserved for format-specific metadata. \par \b Rationale\/ \:

Q: Why Extra.allow for line metadata? A: Different parsers attach different metadata (heading levels, indentation, list numbering). Extra.allow prevents loss. \par \b Changes\/ \:
