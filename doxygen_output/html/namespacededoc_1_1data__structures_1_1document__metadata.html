\par \b Module Contract\/ \:\par \b Purpose\/ \: Define the DocumentMetadata data structure — holds file-level metadata and supports dynamic attribute addition.

\par \b Scope\/ \: Document metadata: file info, timestamps, UID generation, extensible attributes. \par \b Input\/ \: File name, temp name, size, timestamps, MIME type, optional UID, extra kwargs. \par \b Output\/ \: DocumentMetadata instance with auto-generated or provided UID. \par \b Links\/ \: [INHERITS(5): Serializable, USES_API(8): ApiDocumentMetadata] \par \b Invariants\/ \:

  • uid is always a non-empty string (auto-generated if not provided) \par \b Rationale\/ \:

Q: Why use **kwargs for extensibility? A: Different document formats have different metadata fields; kwargs enable format-specific extensions without schema changes. \par \b Changes\/ \:
