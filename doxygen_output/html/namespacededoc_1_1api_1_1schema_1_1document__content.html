\par \b Module Contract\/ \:\par \b Purpose\/ \: Define the DocumentContent schema — the core container for parsed document content: a tree structure of text nodes and a list of tables.

\par \b Scope\/ \: Document content data model — structure and tables. \par \b Input\/ \: None (standalone model, references TreeNode and Table). \par \b Output\/ \: Pydantic BaseModel DocumentContent with structure (TreeNode) and tables (List[Table]). \par \b Links\/ \: [USES_API(7): pydantic.BaseModel; READS_DATA_FROM(8): TreeNode, Table] \par \b Invariants\/ \:

  • structure is always a valid TreeNode (root of document tree).
  • tables list may be empty but never None. \par \b Rationale\/ \:

Q: Why separate structure from tables? A: Tables have fundamentally different layout (grid vs. tree). Separating them allows specialized rendering and querying. \par \b Changes\/ \:
