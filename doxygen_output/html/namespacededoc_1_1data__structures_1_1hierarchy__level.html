\par \b Module Contract\/ \:\par \b Purpose\/ \: Define the HierarchyLevel data structure — two-dimensional importance levels (level_1, level_2) with total ordering for document tree construction.

\par \b Scope\/ \: Hierarchy levels: comparison operators, factory methods for known line types. \par \b Input\/ \: Level_1, level_2 (Optional[int]), multiline flag, line type string. \par \b Output\/ \: HierarchyLevel instances with total ordering support. \par \b Links\/ \: [USED_BY(9): LineMetadata, TreeNode] \par \b Invariants\/ \:

  • level_1 is either None or >= 0
  • level_2 is either None or >= 0
  • None levels are treated as +inf in comparisons \par \b Rationale\/ \:

Q: Why two-level hierarchy (level_1 and level_2)? A: Documents have both primary importance (root > header > list > raw_text) and secondary ordering within the same type (list item numbering). Two-dimensional model captures both aspects. \par \b Changes\/ \:
