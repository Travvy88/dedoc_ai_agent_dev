\par \b Module Contract\/ \:\par \b Purpose\/ \: Define the ParsedDocument schema — the top-level API response model aggregating document content, metadata, attachments, warnings, and version info.

\par \b Scope\/ \: Root API response model — full document parsing result. \par \b Input\/ \: None (standalone model, references DocumentContent, DocumentMetadata, self for attachments). \par \b Output\/ \: Pydantic BaseModel ParsedDocument with content, metadata, version, warnings, attachments. \par \b Links\/ \: [USES_API(7): pydantic.BaseModel; READS_DATA_FROM(8): DocumentContent, DocumentMetadata] \par \b Invariants\/ \:

  • content is always present (may have empty tables and root-only structure).
  • attachments list may be empty if with_attachments=False.
  • version matches dedoc.version.__version__. \par \b Rationale\/ \:

Q: Why self-referential for attachments? A: Attached files are parsed into ParsedDocument themselves, forming a recursive structure that matches the nested nature of document attachments. \par \b Changes\/ \:
