\par \b Module Contract\/ \:\par \b Purpose\/ \: To define the abstract base class for all document format converters, providing a uniform interface (can_convert/convert), shared subprocess execution with timeout safety, and configurable supported format registries.

\par \b Scope\/ \: Converter interface definition, MIME/extension matching, external tool subprocess execution. \par \b Input\/ \: Configuration dict, supported extension/mime sets. \par \b Output\/ \: AbstractConverter ABC for concrete subclasses. \par \b Links\/ \: [USES_API(8): dedoc.utils.utils.get_mime_extension, subprocess; READS_DATA_FROM(7): dedoc.common.exceptions.ConversionError] \par \b Invariants\/ \:

  • can_convert() always returns a boolean.
  • _run_subprocess() always raises ConversionError on failure or timeout.
  • convert() is abstract — concrete subclasses must implement it. \par \b Rationale\/ \:

Q: Why an ABC with shared _run_subprocess instead of standalone utility? A: Each converter may need different timeouts or logging configs. Encapsulating subprocess logic in the base class gives subclasses a consistent, configurable execution harness. \par \b Changes\/ \:
