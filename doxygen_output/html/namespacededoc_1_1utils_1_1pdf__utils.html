\par \b Module Contract\/ \:\par \b Purpose\/ \: To provide PDF-specific utilities for page counting and page-to-image conversion, enabling the Dedoc pipeline to extract visual representations of PDF pages.

\par \b Scope\/ \: PDF page count retrieval, PDF page image rendering. \par \b Input\/ \: File path to PDF document, page index. \par \b Output\/ \: Page count (int), page image (PIL.Image). \par \b Links\/ \: [USES_API(8): pypdf, pdf2image] \par \b Invariants\/ \:

  • get_pdf_page_count returns None (not raises) on unreadable PDFs.
  • get_page_image returns None if no images rendered for the given page. \par \b Rationale\/ \:

Q: Why use pypdf instead of PyPDF2? A: pypdf is the actively maintained successor with better error handling. \par \b Changes\/ \:

LAST_CHANGE: [v1.0.0 – Added SEMANTIC TEMPLATE markup and LDD logging] \par \b Module Map\/ \:

FUNC 7[PDF page count] => get_pdf_page_count FUNC 8[PDF page to PIL image] => get_page_image \par \b Use Cases\/ \:

  • [get_pdf_page_count]: Reader → AssessDocumentLength → Page count reported
  • [get_page_image]: StructureExtractor → ExtractVisualFeatures → Page image rendered
